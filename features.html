What's inside

Every feature, intentionally built.

No framework did this for us. Each capability below was designed, routed, and wired to the database by hand.

⚙

Zero-Framework HTTP Server

Backend Core

The entire request lifecycle — parsing URLs, reading body streams, setting response headers — is handled using only Node's native http module. No Express, no Koa, no Fastify. The router is a hand-rolled pattern matcher that dispatches to the right handler based on method and path.

🔑

bcrypt Password Hashing

Security

Plaintext passwords never touch the database. On registration, the password is salted and hashed using bcrypt before being stored. On login, the provided password is compared against the stored hash — the original is never recoverable or reversible.

🪙

JWT-Based Authentication

Security

Sessions are stateless. After login, the server signs a JSON Web Token containing the user's ID and account number, then sets it as a secure, HTTP-only cookie. Every protected route verifies the token before processing the request — no session store, no global state.

🔮

Native Crypto UUIDs

Security

Account numbers and wallet IDs are generated using Node's built-in node:crypto module — no third-party UUID libraries needed. Each identifier is cryptographically random and unique across the entire database.

🗄

PostgreSQL Relational Ledger

Database

Users and transactions live in separate, related PostgreSQL tables. Every deposit, withdrawal, and transfer writes a ledger entry with a type, amount, timestamp, and reference to the account involved — giving you a full, queryable audit trail.

💸

Fund Transfers

Banking

Transfer funds between accounts by email or wallet ID. The backend validates the recipient, deducts from the sender's balance, credits the recipient, and writes two ledger entries atomically — debit and credit — so the books always balance.

📊

Transaction History

Banking

Each user can view their complete transaction history, ordered by timestamp. Deposits credit green, withdrawals and transfers debit red. The admin panel can also inspect the transaction history of any account in the system.

🛡

Admin Control Panel

Admin

A password-protected admin interface lets you view all registered users, their balances and statuses, and take actions: freeze accounts, unfreeze them, delete users permanently, or drill into any user's transaction log. Stats on total, active, and frozen accounts are computed live.

🧪

Sandbox Economy

Testing

Deposits and withdrawals accept arbitrary values without real-world constraints — intentionally. This lets you flood the transaction layer with test data, verify routing logic, and inspect database writes without any external payment integration in the way.

🍪

Secure Cookie Management

Security

Auth tokens are delivered via HttpOnly cookies — making them inaccessible to JavaScript running on the page and protecting against XSS-based token theft. The server manually sets and clears these cookies as part of the login and logout flows.
